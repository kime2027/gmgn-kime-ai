核心工作流

1. 新币发现与安全评估

```
启动应用
    ↓
初始化所有模块
    ↓
开始扫描循环 (每30秒)
    ├─ 从 GMGN API 获取新币列表
    ├─ 初步过滤 (流动性、持有者数量)
    └─ 对通过的代币进行安全检查
         ├─ Honeypot 检测
         ├─ LP 锁定状态
         ├─ Dev 抛售风险
         └─ 持有者集中度
              ↓
         安全评分 ≥ 60?
         ├─ 是 → 发送通知 + 触发买入
         └─ 否 → 标记为不安全
```

2. 交易执行流程

```
检测到安全新币
    ├─ 计算买入金额
    │  (MIN_BUY_AMOUNT ~ MAX_BUY_AMOUNT)
    ├─ 选择活跃钱包
    ├─ 执行买入交易
    │  ├─ 获取实时价格
    │  └─ 生成交易指令
    ├─ 开立持仓记录
    │  ├─ 记录入场价格
    │  ├─ 计算止盈价格 (500% 上涨)
    │  ├─ 计算止损价格 (50% 下跌)
    │  └─ 初始化最高价格
    └─ 发送 Telegram 通知
```

3. 风险管理流程

```
持仓管理循环 (每60秒)
    ├─ 获取所有活跃持仓
    └─ 对每个持仓:
         ├─ 获取最新价格
         ├─ 更新持仓价值
         ├─ 检查止盈条件
         │  └─ 价格 ≥ 入场价 × (1 + TP%) ?
         │      ├─ 是 → 执行平仓 (止盈)
         │      └─ 否 → 继续监控
         ├─ 检查止损条件
         │  └─ 价格 ≤ 入场价 × (1 - SL%) ?
         │      ├─ 是 → 执行平仓 (止损)
         │      └─ 否 → 继续监控
         └─ 检查跟踪止损
            └─ 价格 ≤ 最高价 × (1 - TSL%) ?
                ├─ 是 → 执行平仓 (跟踪止损)
                └─ 否 → 更新最高价格
```

4. 聪明钱跟单流程

```
定期检查聪明钱 (每2小时)
    ├─ 搜索盈利钱包 (收益 > 100%)
    ├─ 获取钱包最新持仓
    └─ 对每个持仓:
         ├─ 检查是否已持有
         │  └─ 已持有 → 跳过
         ├─ 获取代币信息
         ├─ 流动性检查
         │  └─ 流动性 ≥ MIN_LIQUIDITY ?
         │      ├─ 是 → 执行跟进买入
         │      └─ 否 → 跳过
         └─ 创建持仓记录
```

数据流向

API 调用流程

```
GMGNAPIClient
    │
    ├─ get_new_tokens()          [获取新币列表]
    ├─ get_token_info()          [获取代币详情]

    ├─ get_liquidity_info()      [获取流动性]
    ├─ get_token_trades()        [获取交易历史]
    ├─ get_smart_money_trades()  [获取聪明钱交易]
    ├─ get_wallet_info()         [获取钱包信息]
    ├─ get_wallet_portfolio()    [获取钱包持仓]
    └─ search_smart_money()      [搜索聪明钱]
```

持仓数据生命周期

```
创建持仓
    ├─ 参数: token_addr, 入场价, 买入金额
    ├─ 初始化: 数量, 成本基数, TP/SL 价格
    └─ 状态: OPEN
        │
        ├─ 价格更新 (每60秒)
        │   ├─ 更新当前价格
        │   ├─ 计算当前价值
        │   └─ 更新最高价格
        │
        ├─ 检查平仓条件
        │   ├─ 止盈 → 平仓 (状态: CLOSED)
        │   ├─ 止损 → 平仓 (状态: CLOSED)
        │   └─ 跟踪止损 → 平仓 (状态: CLOSED)
        │
        └─ 计算最终结果
            ├─ 入场: $100
            ├─ 退出: $600
            ├─ 盈亏: +$500 (+500%)
            └─ 归档到历史记录
```

安全检查详解

Honeypot 检测

```
检查项目:
├─ is_honeypot 标志
├─ can_transfer 功能
├─ blacklist_enabled 状态

评分规则:
├─ 确实是 honeypot → 0 分
├─ 转账受限 → 30 分
├─ 启用黑名单 → 50 分
└─ 正常 → 100 分
```

LP 锁定检查

```
检查项目:
├─ is_locked (LP 是否锁定)
├─ lock_days (锁定天数)

评分规则:
├─ 未锁定 → 30 分
├─ < 30 天 → 50 分
├─ ≥ 365 天 → 100 分
└─ 其他 → 50 + (lock_days/365)  50
```

Dev 抛售风险

```
检查项目:
├─ 识别 dev 钱包
├─ 计算 dev 持仓比例

评分规则:
├─ > 50% → 20 分 (风险极高)
├─ 30-50% → 60 分 (中等风险)
└─ < 30% → 100 分 (低风险)
```

持有者集中度

```
检查项目:
├─ 前10大持有者总占比

评分规则:
├─ > MAX_HOLDER_CONCENTRATION → 40 分
├─ 50-MAX% → 70 分
└─ < 50% → 100 分
```

综合评分

```
最终评分 = 100  
    (Honeypot评分/100)  
    (LP评分/100)  
    (Dev评分/100)  
    (集中度评分/100)

判断: 评分 ≥ 60 → 安全 ✅
      评分 < 60 → 不安全 ❌
```

事件触发清单

全局事件

| 事件 | 触发条件 | 动作 |
|------|---------|------|
| 应用启动 | 主程序运行 | 初始化所有模块、连接 Telegram |
| 应用关闭 | Ctrl+C 或错误 | 关闭所有任务、断开连接 |
| 配置错误 | 缺少必要配置 | 报错退出 |

扫描事件

| 事件 | 触发条件 | 动作 |
|------|---------|------|
| 发现新币 | API 返回新代币 | 标记为未检查 |
| 通过初筛 | 流动性 ≥ 最小值 | 进行安全检查 |
| 安全评分高 | 评分 ≥ 60 | 发送发现通知 + 触发买入 |

交易事件

| 事件 | 触发条件 | 动作 |
|------|---------|------|
| 执行买入 | 代币通过检查 | 创建持仓、发送通知 |
| 止盈触发 | 价格达到目标 | 执行平仓、记录收益 |
| 止损触发 | 价格低于止损 | 执行平仓、止损 |
| 跟踪止损 | 价格回落幅度 | 执行平仓、保留收益 |

钱包事件

| 事件 | 触发条件 | 动作 |
|------|---------|------|
| 余额不足 | 余额 < 买入金额 | 切换钱包或跳过 |
| 交易失败 | 链上交易失败 | 记录错误、禁用钱包 |
| 余额更新 | 定期同步 | 更新钱包余额 |

性能考虑

扫描性能

- 扫描间隔: 30秒
- 批处理大小: 100个代币
- API 调用限制: 考虑速率限制

监控性能

- 检查间隔: 60秒
- 持仓数量: 可能同时管理多个
- API 调用: ~1次/持仓/检查周期

存储优化

- 日志文件: 按大小轮转
- 数据库: SQLite 本地存储
- 缓存: Redis (可选)

错误处理

恢复机制

```
API 连接失败
    ├─ 记录错误
    ├─ 等待 5 秒
    └─ 重试 (最多 3 次)

交易执行失败
    ├─ 记录错误
    ├─ 发送错误通知
    └─ 禁用相关钱包

价格获取失败
    ├─ 使用缓存价格
    ├─ 延迟检查
    └─ 继续监控

Telegram 连接失败
    ├─ 记录警告
    └─ 继续运行 (无通知)
```
