📊 贡献指南

开发优先级

1. 高优先级
   - 安全相关的改进
   - 严重 Bug 修复
   - 性能关键优化
   - 核心功能完善

2. 中优先级
   - 新交易策略
   - UX 改进
   - 文档完善
   - 测试增强

3. 低优先级
   - 小功能请求
   - 代码风格改进
   - 性能微优化
   - 示例项目

如何贡献

```bash
1. Fork 项目
2. 创建功能分支
git checkout -b feature/your-feature

3. 提交代码
git add .
git commit -m "feat: add your feature"

4. 推送分支
git push origin feature/your-feature

5. 创建 Pull Request
```

代码标准

- Python 3.11+
- PEP 8 编码风格
- Type hints
- 单元测试覆盖 > 80%
- 文档完整

🐛 已知问题

当前限制

1. API 依赖
   - 完全依赖 GMGN OpenAPI
   - 无备用数据源
   - 速率限制约束

2. 安全检查
   - 无法 100% 检测 honeypot
   - LP 分析基于已知特征
   - Dev 风险评估相对简单

3. 交易执行
   - 仅支持 Solana 链
   - Gas 费用不够精确
   - Slippage 控制有限

4. 监控覆盖
   - 仅监控 GMGN 追踪的币
   - 无法快于 API 更新频率
   - 历史数据有限

改进计划

- [ ] 多数据源支持
- [ ] 本地链上验证
- [ ] 链上交易执行
- [ ] 增量数据缓存
